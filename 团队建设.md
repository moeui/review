## 你如何推动前端团队的技术统一（如代码规范、工程化）？

- 建立共识机制：技术评审会、文档沉淀达成一致
- 制定 ESLint + Prettier 规范，并通过 Git Hooks 强制校验
- 定期 code review，pr 审核等

## 当团队成员对技术方案有分歧时，你怎么处理？

- 尊重 + 数据驱动：不强行推行，用性能数据、开发效率、可维护性等指标说服。

## 如何与产品、后端高效协作？

- 跨团队协作：主动对齐接口规范（如 Swagger）、联调流程、上线节奏；定期同步进度。

## 如果组员技术能力参差不齐，怎么安排任务？

- 先修 bug，再做简单 feature，最后参与模块设计（给新人明确的小任务 + 文档 + Mentor；给 senior 挑战性任务）
- 鼓励分享，共同成长

## 如何做 Code Review 才有效？

- 聚焦可读性、可维护性、性能、安全
- 用“建议”代替“命令”（如：“这里是否考虑用 useMemo 优化？”）
- 建立 Checklist（如：边界 case、错误处理、单元测试）
